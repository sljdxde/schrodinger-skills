# Data Source Playbook

本文件用于执行购房分析的数据采集。采集时优先找原始来源，多源交叉验证，最后用证据台账承载不确定性。

## 证据台账

每条关键结论都用这 6 列自查：

| 字段 | 写法 |
|---|---|
| 事实/数据 | 例如“近一年主力成交单价约 X 万/㎡” |
| 来源 | 网站、机构、文件名或平台 |
| 时间 | 发布日期或访问日期 |
| 适用范围 | 城市、片区、楼盘、户型、年份 |
| 置信度 | 高/中/低 |
| 备注 | 样本量、口径差异、是否挂牌价 |

## 数据源优先级

1. 政府与官方：统计局、教育局、住建局、规划资源局、民政/街道公开信息、学校官网或公众号。
2. 交易与市场：贝壳/链家成交、安居客、房天下、诸葛找房、克而瑞、中指研究院、本地网签平台。
3. 地理与配套：高德/百度地图、地铁官网、医院/商圈/公园官方信息。
4. 舆情与口碑：主流媒体、自媒体、论坛、问答、业主讨论。只能做辅助线索。

## 搜索关键词

### 楼盘和交易

- `{城市} {小区名} 成交价`
- `{城市} {小区名} 贝壳 成交`
- `{小区名} 挂牌价 成交价`
- `{片区名} 二手房 成交量 均价`
- `{小区名} 房价 走势图`
- `{城市} 二手房 网签 月报`

### 政策和城市基本面

- `{城市} 统计公报 常住人口 出生率`
- `{城市} 住宅用地 供应 规划`
- `{城市} 限购 限贷 公积金 最新`
- `{城市} 二手房 成交量 月报`

### 可比样本

可比楼盘至少满足 2 个条件：

- 同城市同板块，或通勤/学校诉求相近。
- 总价带相近。
- 建成年代、产品类型、居住品质接近。
- 同一学校梯队或教育预期相近。

不要拿“纯学区老破小”和“品质改善次新房”直接比单价；必须拆成教育溢价、居住价值、流动性三个维度解释。

### 学区溢价样本

涉及学区房时，还必须按 `school-premium-comparison.md` 选择 2-3 个周边非学区或弱学区可比盘，用于估算目标房源相对普通居住价值的教育溢价。不要只做同学区内部小区比较。

## 数据处理规则

- 成交价优先于挂牌价；挂牌价只能反映卖方预期。
- 样本少于 5 套时，不要给精确均价，只给区间和样本说明。
- 同一小区必须按面积段、楼层、装修和户型拆分；学区房常见小户型单价高于大户型。
- 近 3 个月数据适合判断热度，近 12-36 个月适合判断趋势。
- 遇到平台数据冲突时，列出口径差异，不强行合并成一个数字。

## 输出前检查

- 是否标注了所有核心价格的来源和时间？
- 是否区分了成交、挂牌、网签、评估和传闻？
- 是否说明样本量和户型面积段？
- 是否有 2-3 个真正可比对象？
- 如果是学区房，是否有周边非学区/弱学区可比盘和溢价率解释？
- 是否把低可信口碑和高可信官方数据分开使用？
