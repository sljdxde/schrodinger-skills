# Intake Questionnaire

本文件定义 `house-buying` 的交互式启动规则。默认先问清楚用户需求，再采集数据和生成报告。

## 目标

避免在预算、目标小区、学校诉求或入学时间缺失时直接跑分析。购房结论高度依赖家庭约束，信息不完整时必须先追问。

## 字段分级

### 必填字段

| 字段 | 说明 | 缺失时处理 |
|---|---|---|
| 城市 | 目标城市 | 必须追问 |
| 目标对象 | 小区、楼盘、学校或片区 | 必须追问 |
| 购房目的 | 自住、学区、改善、投资或组合 | 必须追问 |
| 预算 | 总价或单价预算 | 必须追问 |
| 入学年份 | 孩子预计上小学/初中的年份；不涉及学区可填“无” | 学区相关必须追问 |

### 可选字段

| 字段 | 说明 | 缺失时处理 |
|---|---|---|
| 户型/面积 | 几房、面积段、楼层、房龄偏好 | 可默认“未指定” |
| 房票/贷款约束 | 首套/二套、首付、贷款、月供上限 | 可默认“未指定” |
| 通勤与配套 | 上班地点、老人照护、地铁/医院/商圈偏好 | 可默认“未指定” |
| 决策时间 | 马上、3 个月、半年、1 年以上 | 可默认“观望” |
| 备选项 | 备选小区、学校、片区或明确排除项 | 可默认“无明确备选” |

## 首轮提问模板

用户信息不足时，使用下面的问题一次性收集，最多 8 个：

```text
为了避免信息缺失导致误判，我先确认几个关键约束：

1. 目标城市是哪里？
2. 你想分析的小区、楼盘、学校或片区是什么？
3. 购房目的是什么：自住、学区、改善、投资，还是组合？
4. 总预算或单价预算是多少？
5. 孩子预计哪一年上小学/初中？如果只是自住可填“无”。
6. 目标户型、面积、楼层或房龄偏好是什么？
7. 计划多久内决策：马上、3 个月、半年、1 年以上？
8. 有没有备选小区或明确排除项？
```

## 追问策略

- 只给城市和楼盘，例如“分析杭州耀江文鼎苑”：追问预算、购房目的、入学年份和决策时间。
- 只给城市和泛需求，例如“想买杭州学区房”：先要求缩小到目标学校、片区或 2-3 个候选小区。
- 已给关键字段，例如“杭州耀江文鼎苑，自住+学区，预算 400 万，孩子 2029 年上小学”：直接开始分析，缺失的户型、贷款和备选项列为假设。
- 用户拒绝补充某项信息：继续分析，但在报告中标注“用户未提供”，并降低相关结论置信度。
- 用户目标明显冲突，例如“投资为主但不接受价格波动”：先指出冲突并要求确认优先级。

## 进入分析的门槛

满足以下条件才开始完整数据采集：

- 城市明确。
- 目标对象明确到小区、楼盘、学校、片区或候选列表。
- 购房目的明确。
- 预算明确，或用户明确表示“先不设预算”。
- 学区相关需求必须明确入学年份；非学区需求可写“无”。

不满足门槛时，不输出完整报告，只输出追问清单。
